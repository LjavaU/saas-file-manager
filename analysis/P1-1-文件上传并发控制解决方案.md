# P1-1: 文件上传无全局并发控制 - 解决方案

**风险描述**: 多用户同时批量上传时，缺少全局并发控制，可能导致 MinIO 连接池耗尽、网络带宽打满

**影响面**: 性能 + 稳定性

**触发条件**: 50 个用户同时上传，单次 20 个文件 → 1000 个文件并发处理

---

## 一、解决方案（3 个层次）

### 🎯 方案一：全局并发限流（推荐 ⭐⭐⭐⭐⭐）

#### **核心思路**
使用 **Guava RateLimiter** 或 **Semaphore** 控制全局上传并发数

#### **实施方案**

##### 1. 在 FileManagerImpl 中添加全局信号量

```java
package com.supcon.tptrecommend.manager.impl;

import java.util.concurrent.Semaphore;
import java.util.concurrent.TimeUnit;

@Service
@Slf4j
public class FileManagerImpl implements FileManager {

    // ===== 新增：全局上传并发限制 =====
    /**
     * 全局上传并发限制：最多同时处理 100 个文件上传
     * 根据 MinIO 连接池大小和服务器带宽调整此值
     */
    private final Semaphore uploadSemaphore = new Semaphore(100);

    /**
     * 上传超时时间（秒）
     */
    private static final int UPLOAD_ACQUIRE_TIMEOUT = 5;

    // ... 其他字段
}
```

##### 2. 修改 upload() 方法

```java
/**
 * 上传文件
 *
 * @param file 文件
 * @param path 路径
 * @return {@link FileObjectResp}
 */
@Override
public FileObjectResp upload(MultipartFile file, String path) {
    LoginInfoUserDTO user = LoginUserUtils.getLoginUserInfo();

    // ===== 新增：获取上传许可 =====
    boolean acquired = false;
    try {
        acquired = uploadSemaphore.tryAcquire(UPLOAD_ACQUIRE_TIMEOUT, TimeUnit.SECONDS);
        if (!acquired) {
            log.warn("用户 {} 上传文件被限流，当前并发已达上限", user.getUsername());
            throw new ClientException("系统繁忙，当前上传请求过多，请稍后重试");
        }

        // 原有上传逻辑
        FileObjectResp fileObjectResp = doUpload(file, path, user);
        doFileProcess(fileObjectResp.getId(), user.getId(), fileObjectResp.getOriginalName(), null);
        return fileObjectResp;

    } catch (InterruptedException e) {
        Thread.currentThread().interrupt();
        throw new ServerException("上传请求被中断");
    } finally {
        // ===== 新增：释放许可 =====
        if (acquired) {
            uploadSemaphore.release();
        }
    }
}
```

##### 3. 修改 batchUpload() 方法

```java
/**
 * 批量上传
 *
 * @param multipartFiles 多部分文件
 * @param path           路径
 * @return {@link List}<{@link FileObjectResp}>
 */
@Override
public List<FileObjectResp> batchUpload(List<MultipartFile> multipartFiles, String path) {
    if (CollUtil.isEmpty(multipartFiles)) {
        throw new ServerException("上传文件不能为空");
    }
    // 限制文件为20个
    if (multipartFiles.size() > 20) {
        throw new ClientException("一次最多上传20个文件");
    }

    LoginInfoUserDTO loginUser = LoginUserUtils.getLoginUserInfo();
    String currentTenant = TenantContext.getCurrentTenant();

    // ===== 新增：批量获取许可（每个文件需要一个许可）=====
    int fileCount = multipartFiles.size();
    boolean acquired = false;
    try {
        acquired = uploadSemaphore.tryAcquire(fileCount, UPLOAD_ACQUIRE_TIMEOUT, TimeUnit.SECONDS);
        if (!acquired) {
            log.warn("用户 {} 批量上传 {} 个文件被限流", loginUser.getUsername(), fileCount);
            throw new ClientException("系统繁忙，请减少上传文件数量或稍后重试");
        }

        // 原有批量上传逻辑
        List<CompletableFuture<FileObjectResp>> futures = multipartFiles.stream()
            .map(file -> CompletableFuture.supplyAsync(() -> {
                TenantContext.setCurrentTenant(currentTenant);
                try {
                    return doUpload(file, path, loginUser);
                } finally {
                    TenantContext.clear();
                }
            }, fileUploadExecutor))
            .collect(Collectors.toList());

        CompletableFuture<Void> allOf = CompletableFuture.allOf(futures.toArray(new CompletableFuture[0]));

        List<FileObjectResp> fileObjectResps = allOf.thenApply(v -> futures.stream()
                .map(CompletableFuture::join)
                .collect(Collectors.toList()))
            .join();

        // 创建文件处理任务
        fileObjectResps.forEach(fileResp -> doFileProcess(fileResp.getId(), loginUser.getId(), fileResp.getOriginalName(), null));
        return fileObjectResps;

    } catch (InterruptedException e) {
        Thread.currentThread().interrupt();
        throw new ServerException("批量上传请求被中断");
    } finally {
        // ===== 新增：释放许可 =====
        if (acquired) {
            uploadSemaphore.release(fileCount);
        }
    }
}
```

#### **优点**

- ✅ 保护后端资源（MinIO、网络带宽）

- ✅ 无需引入额外依赖

#### **缺点**
- ❌ 所有用户共享配额，可能被少数用户占满
- ❌ 无法按租户/用户维度限流

### 🎯 方案二：租户级并发控制（更精细 ⭐⭐⭐⭐）

#### **核心思路**
按租户/用户维度限流，防止单个租户占用全部资源

#### **实施方案**

##### 1. 添加依赖（如果项目中没有 Guava）


##### 2. 在 FileManagerImpl 中添加租户级限流器

```java
import com.google.common.cache.CacheBuilder;
import com.google.common.cache.CacheLoader;
import com.google.common.cache.LoadingCache;

@Service
@Slf4j
public class FileManagerImpl implements FileManager {

    // ===== 新增：租户级上传限流器 =====
    /**
     * 租户级上传并发限制
     * Key: tenantId
     * Value: 该租户的 Semaphore（最多 20 个并发）
     */
    private final LoadingCache<String, Semaphore> tenantUploadLimiters =
        CacheBuilder.newBuilder()
            .maximumSize(1000)  // 最多缓存 1000 个租户
            .expireAfterAccess(10, TimeUnit.MINUTES)  // 10 分钟无访问自动清理
            .build(new CacheLoader<String, Semaphore>() {
                @Override
                public Semaphore load(String tenantId) {
                    log.info("为租户 {} 创建上传限流器，并发上限: 20", tenantId);
                    return new Semaphore(20); // 每租户最多 20 个并发上传
                }
            });

    /**
     * 租户级上传超时时间（秒）
     */
    private static final int TENANT_UPLOAD_TIMEOUT = 3;
}
```

```java
/**

public Map<String, Object> getTenantUploadStatus(String tenantId) {
    Semaphore limiter = tenantUploadLimiters.getIfPresent(tenantId);
    if (limiter == null) {
        return Map.of("message", "租户暂无上传活动");
    }

    Map<String, Object> status = new HashMap<>();
    status.put("tenantId", tenantId);
    status.put("availablePermits", limiter.availablePermits());
    status.put("queueLength", limiter.getQueueLength());
    status.put("maxConcurrency", 20);
    status.put("currentConcurrency", 20 - limiter.availablePermits());
    return status;
}

/**
 * 获取所有租户上传状态汇总
 */
public List<Map<String, Object>> getAllTenantsUploadStatus() {
    return tenantUploadLimiters.asMap().entrySet().stream()
        .map(entry -> getTenantUploadStatus(entry.getKey()))
        .collect(Collectors.toList());
}
```

### 🎯 方案三：基于 Token Bucket 的令牌桶限流（最灵活 ⭐⭐⭐⭐）

#### **核心思路**
使用 **Resilience4j RateLimiter** 或 **Bucket4j** 实现平滑限流

#### **实施方案**

##### 1. 引入依赖

```xml
<dependency>
    <groupId>io.github.resilience4j</groupId>
    <artifactId>resilience4j-ratelimiter</artifactId>
    <version>2.1.0</version>
</dependency>
<dependency>
    <groupId>io.github.resilience4j</groupId>
    <artifactId>resilience4j-spring-boot2</artifactId>
    <version>2.1.0</version>
</dependency>
```

##### 2. 在 application.yml 中配置限流规则

```yaml
resilience4j:
  ratelimiter:
    instances:
      file-upload:
        limit-for-period: 500        # 每个周期最多 500 次调用
        limit-refresh-period: 60s    # 60 秒刷新一次
        timeout-duration: 5s         # 等待超时 5 秒
        register-health-indicator: true
        event-consumer-buffer-size: 100
```

##### 3. 创建限流配置类

```java
package com.supcon.tptrecommend.config;

import io.github.resilience4j.ratelimiter.RateLimiter;
import io.github.resilience4j.ratelimiter.RateLimiterConfig;
import io.github.resilience4j.ratelimiter.RateLimiterRegistry;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

import java.time.Duration;

@Configuration
public class RateLimiterConfiguration {

    @Bean
    public RateLimiterRegistry rateLimiterRegistry() {
        RateLimiterConfig config = RateLimiterConfig.custom()
            .limitForPeriod(500)                            // 每分钟 500 次
            .limitRefreshPeriod(Duration.ofMinutes(1))      // 1 分钟刷新
            .timeoutDuration(Duration.ofSeconds(5))         // 等待超时 5 秒
            .build();

        return RateLimiterRegistry.of(config);
    }

    @Bean
    public RateLimiter fileUploadRateLimiter(RateLimiterRegistry registry) {
        return registry.rateLimiter("file-upload");
    }
}
```

##### 4. 在 FileManagerImpl 中应用限流器

```java
import io.github.resilience4j.ratelimiter.RateLimiter;
import io.github.resilience4j.ratelimiter.RequestNotPermitted;

@Service
@Slf4j
public class FileManagerImpl implements FileManager {

    private final RateLimiter fileUploadRateLimiter;

    public FileManagerImpl(
        // ... 其他参数
        RateLimiter fileUploadRateLimiter
    ) {
        // ... 其他赋值
        this.fileUploadRateLimiter = fileUploadRateLimiter;
    }

    @Override
    public FileObjectResp upload(MultipartFile file, String path) {
        try {
            // ===== 使用 RateLimiter 装饰上传方法 =====
            return RateLimiter.decorateCheckedSupplier(
                fileUploadRateLimiter,
                () -> {
                    LoginInfoUserDTO user = LoginUserUtils.getLoginUserInfo();
                    FileObjectResp fileObjectResp = doUpload(file, path, user);
                    doFileProcess(fileObjectResp.getId(), user.getId(),
                                  fileObjectResp.getOriginalName(), null);
                    return fileObjectResp;
                }
            ).apply();

        } catch (RequestNotPermitted e) {
            log.warn("上传限流触发：当前请求速率过高");
            throw new ClientException("系统繁忙，当前请求过多（速率限制），请稍后重试");
        } catch (Throwable throwable) {
            throw new ServerException("上传失败", throwable);
        }
    }
}
```

## 二、🎯 最终推荐方案：方案一 + 方案二 组合

### **双重限流架构**

结合全局限流和租户级限流，提供最佳的资源保护和公平性。

#### **实施代码**

```java
package com.supcon.tptrecommend.manager.impl;

import com.google.common.cache.CacheBuilder;
import com.google.common.cache.CacheLoader;
import com.google.common.cache.LoadingCache;

@Service
@Slf4j
public class FileManagerImpl implements FileManager {

    // ===== 双重限流配置 =====

    /**
     * 全局上传并发限制：最多同时处理 100 个文件上传
     * 防止系统资源耗尽
     */
    private final Semaphore globalUploadSemaphore = new Semaphore(100);

    /**
     * 租户级上传限流器
     * 每个租户最多 20 个并发，防止单租户占用过多资源
     */
    private final LoadingCache<String, Semaphore> tenantUploadLimiters =
        CacheBuilder.newBuilder()
            .maximumSize(1000)
            .expireAfterAccess(10, TimeUnit.MINUTES)
            .build(new CacheLoader<String, Semaphore>() {
                @Override
                public Semaphore load(String tenantId) {
                    return new Semaphore(20);
                }
            });

    private static final int GLOBAL_TIMEOUT = 2;  // 全局超时 2 秒
    private static final int TENANT_TIMEOUT = 3;  // 租户超时 3 秒

    // ===== 双重限流的上传方法 =====

    @Override
    public FileObjectResp upload(MultipartFile file, String path) {
        LoginInfoUserDTO user = LoginUserUtils.getLoginUserInfo();
        String tenantId = TenantContext.getCurrentTenant();

        // 步骤1：先检查租户配额
        Semaphore tenantLimiter = tenantUploadLimiters.getUnchecked(tenantId);
        boolean tenantAcquired = false;
        boolean globalAcquired = false;

        try {
            tenantAcquired = tenantLimiter.tryAcquire(TENANT_TIMEOUT, TimeUnit.SECONDS);
            if (!tenantAcquired) {
                log.warn("租户 {} 用户 {} 上传被限流（租户配额满）", tenantId, user.getUsername());
                throw new ClientException("当前租户上传请求过多，请稍后重试");
            }

            // 步骤2：再检查全局配额
            globalAcquired = globalUploadSemaphore.tryAcquire(GLOBAL_TIMEOUT, TimeUnit.SECONDS);
            if (!globalAcquired) {
                log.warn("租户 {} 用户 {} 上传被限流（全局配额满）", tenantId, user.getUsername());
                throw new ClientException("系统繁忙，请稍后重试");
            }

            // 执行上传
            FileObjectResp fileObjectResp = doUpload(file, path, user);
            doFileProcess(fileObjectResp.getId(), user.getId(),
                          fileObjectResp.getOriginalName(), null);
            return fileObjectResp;

        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
            throw new ServerException("上传请求被中断");
        } finally {
            // 释放许可（注意顺序：先释放全局，再释放租户）
            if (globalAcquired) {
                globalUploadSemaphore.release();
            }
            if (tenantAcquired) {
                tenantLimiter.release();
            }
        }
    }

    @Override
    public List<FileObjectResp> batchUpload(List<MultipartFile> multipartFiles, String path) {
        if (CollUtil.isEmpty(multipartFiles)) {
            throw new ServerException("上传文件不能为空");
        }
        if (multipartFiles.size() > 20) {
            throw new ClientException("一次最多上传20个文件");
        }

        LoginInfoUserDTO loginUser = LoginUserUtils.getLoginUserInfo();
        String tenantId = TenantContext.getCurrentTenant();
        String currentTenant = tenantId;
        int fileCount = multipartFiles.size();

        Semaphore tenantLimiter = tenantUploadLimiters.getUnchecked(tenantId);
        boolean tenantAcquired = false;
        boolean globalAcquired = false;

        try {
            // 步骤1：获取租户级许可
            tenantAcquired = tenantLimiter.tryAcquire(fileCount, TENANT_TIMEOUT, TimeUnit.SECONDS);
            if (!tenantAcquired) {
                log.warn("租户 {} 批量上传 {} 个文件被限流（租户配额满）", tenantId, fileCount);
                throw new ClientException("当前租户上传请求过多，请减少文件数量或稍后重试");
            }

            // 步骤2：获取全局许可
            globalAcquired = globalUploadSemaphore.tryAcquire(fileCount, GLOBAL_TIMEOUT, TimeUnit.SECONDS);
            if (!globalAcquired) {
                log.warn("租户 {} 批量上传 {} 个文件被限流（全局配额满）", tenantId, fileCount);
                throw new ClientException("系统繁忙，请减少文件数量或稍后重试");
            }

            // 执行批量上传
            List<CompletableFuture<FileObjectResp>> futures = multipartFiles.stream()
                .map(file -> CompletableFuture.supplyAsync(() -> {
                    TenantContext.setCurrentTenant(currentTenant);
                    try {
                        return doUpload(file, path, loginUser);
                    } finally {
                        TenantContext.clear();
                    }
                }, fileUploadExecutor))
                .collect(Collectors.toList());

            CompletableFuture<Void> allOf = CompletableFuture.allOf(
                futures.toArray(new CompletableFuture[0])
            );

            List<FileObjectResp> fileObjectResps = allOf.thenApply(v ->
                futures.stream()
                    .map(CompletableFuture::join)
                    .collect(Collectors.toList())
            ).join();

            fileObjectResps.forEach(fileResp ->
                doFileProcess(fileResp.getId(), loginUser.getId(),
                              fileResp.getOriginalName(), null)
            );
            return fileObjectResps;

        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
            throw new ServerException("批量上传请求被中断");
        } finally {
            if (globalAcquired) {
                globalUploadSemaphore.release(fileCount);
            }
            if (tenantAcquired) {
                tenantLimiter.release(fileCount);
            }
        }
    }
}
```

### 1. 优化线程池配置

修改 `application-local.yml`：

```yaml
threadpool:

    core-pool-size: 4          # 核心线程 4（与 CPU 核心数一致）
    max-pool-size: 8           # 最大线程 8（2 倍 CPU 核心数）
```

### 5. 前端配合：本地上传队列

前端可以配合实现本地上传队列：

```javascript
// 前端示例代码
class UploadQueue {
    constructor(maxConcurrent = 3) {
        this.maxConcurrent = maxConcurrent;
        this.queue = [];
        this.running = 0;
    }

    async add(file) {
        return new Promise((resolve, reject) => {
            this.queue.push({ file, resolve, reject });
            this.process();
        });
    }

    async process() {
        while (this.running < this.maxConcurrent && this.queue.length > 0) {
            const { file, resolve, reject } = this.queue.shift();
            this.running++;

            try {
                const result = await this.uploadFile(file);
                resolve(result);
            } catch (error) {
                // 如果是 429 错误，延迟重试
                if (error.code === 429) {
                    this.queue.unshift({ file, resolve, reject });
                    await this.sleep(2000); // 等待 2 秒
                } else {
                    reject(error);
                }
            } finally {
                this.running--;
                this.process(); // 继续处理队列
            }
        }
    }

    async uploadFile(file) {
        // 调用后端上传接口
        return axios.post('/open-api/file/upload', formData);
    }

    sleep(ms) {
        return new Promise(resolve => setTimeout(resolve, ms));
    }
}

// 使用
const uploadQueue = new UploadQueue(3);
uploadQueue.add(file1);
uploadQueue.add(file2);
```

## 八、FAQ

**Q1: 为什么选择 Semaphore 而不是 RateLimiter？**

A: Semaphore 更适合并发控制（限制同时进行的操作数），RateLimiter 更适合速率控制（限制单位时间内的请求数）。文件上传是长时间操作，需要控制并发数而非请求速率。

**Q2: 租户级限流器会不会占用太多内存？**

A: 单个 Semaphore 对象约 **200 字节**，1000 个租户仅占用约 **200KB** 内存，且有过期清理机制，影响可忽略。

**Q3: 如果全局配额满了，但某个租户还有配额怎么办？**

A: 这是设计的权衡。全局限流保护系统资源，优先级高于租户配额。可以考虑调大全局配额或引入优先级队列。

**Q4: 如何支持 VIP 租户更高配额？**

A: 修改 `tenantUploadLimiters` 的加载逻辑，从配置中心或数据库读取租户配额配置：

```java
private final LoadingCache<String, Semaphore> tenantUploadLimiters =
    CacheBuilder.newBuilder()
        .build(new CacheLoader<String, Semaphore>() {
            @Override
            public Semaphore load(String tenantId) {
                // 从配置中心读取租户配额（默认 20）
                int quota = tenantConfigService.getUploadQuota(tenantId, 20);
                return new Semaphore(quota);
            }
        });
```

**Q5: 批量上传失败后，已获取的许可如何处理？**

A: 在 `finally` 块中释放，确保即使异常也会归还许可。代码中已正确实现。
